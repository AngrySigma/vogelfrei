# Cleric class kit

Finishes a Cleric after the core `character-generation` workflow. Works on
the same character JSON (`--file`); `$S` =
`.claude/skills/character-generation/scripts`.

Stop if the JSON's `class` is not `Cleric`.

Read these before recording anything:
`docs/Character/Classes/Cleric/index.md` (class grants),
`docs/Character/Classes/Cleric/Special Rules/Blessings.md` (the d20 blessing
table and how chanting works), `docs/Character/Classes/Cleric/Special
Rules/Miracles.md` (for later levels), and the career page (`career_page`,
whose table gives the miracle slots).

## 1. Verify the class grant

- **No combat bonus.** WS +0 / BS +0; `apply_class.py` leaves both at 0.
- **Forced Order.** Alignment must be `Order`
  (docs/Character/Alignment.md); `apply_class.py` enforces it. A Cleric who
  acts against their deity's alignment risks losing divine power until they
  atone — worth a sentence to the player.
- **Read/Write Talent** from the class page.
- Clerics *may* wear armour and use shields (unlike Magic-Users), so no ban
  warning is expected from `finalize.py`.

## 2. The level-1 rule: blessings, not miracles

This is the easy thing to get wrong. On the Priest table level 1 shows
**no miracle slots at all** — the **Invoke Talent** that unlocks prepared
miracles only arrives at level 2. A first-level Cleric therefore casts **no
miracles**; their divine power is expressed entirely through **Blessings**
(class grant: **Blessing Talent**). Record it plainly:

```
python3 $S/annotate.py --file ... --note "Level 1: no miracles yet (Invoke Talent comes at L2). Divine power is Blessings only."
```

## 3. Record the three known blessings

Every Cleric knows **three** blessings granted by their god
(Blessings.md, d20 table). Roll three (or let the player pick to fit the
deity's portfolio — a war-god leans Battle/Might/Courage, a healer
Healing/Recuperation/Hardiness):

```
python3 - <<'PY'
import random
b = ["Battle","Breath","Charisma","Courage","Finesse","Fortune","Grace",
     "Hardiness","Healing","Hunt","Might","Protection","Recuperation",
     "Resolve","Righteousness","Savagery","Tenacity","Wisdom","Wit","Zeal"]
print("\n".join(random.sample(b, 3)))
PY
```

Record them with the effects from the table, and the chant mechanic:

```
python3 $S/annotate.py --file ... \
    --note "Blessings known: <A> (<effect>), <B> (<effect>), <C> (<effect>)" \
    --note "To chant a blessing roll d6; on a 1 it fails and no blessing or miracle works for the rest of the day"
```

## 4. Holy symbol and starting possessions

Invoking divine power needs a **holy symbol clutched in hand** and free
speech (Miracles.md) — make sure the kit has one, or note that it must be
acquired. The class page also lists a bare **"1GC"** it never defines;
record it verbatim and flag it rather than guessing:

```
python3 $S/annotate.py --file ... \
    --note "Needs a holy symbol in hand to invoke (buy or note as owed)" \
    --note "Class page also lists '1GC' — undefined shorthand in the current rulebook; confirm with the Referee"
```

## 5. Career specifics

| Career | Setup |
| --- | --- |
| Priest | Ordained minister; full table. Blessing Talent now, Invoke Talent (first miracle slot) at level 2. Nothing more to add at level 1. |
| Zealot / Warrior Priest | Career pages are blank stubs (no Status, no traits). Record the concept as a note and tell the player the mechanics are pending rulebook content. |

**Status is blank on every Cleric career page**, so `apply_class.py` will
have stopped for `--status`. An ordained Priest is respectable and funded —
Gold or Silver depending on the temple; note which you supplied.

## 6. Re-render

```
python3 $S/finalize.py --file ...        # only if equipment changed
python3 $S/render_sheet.py --file ... --out characters/<slug>.md
```

Blessings and talents render under the sheet's Notes section. When
presenting the character, be clear about the level-1 shape: a devout figure
who leans on three blessings and a holy symbol, with true miracles still one
level away.

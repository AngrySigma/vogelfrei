# Magic-User class kit

Finishes a Magic-User after the core `character-generation` workflow. Works
on the same character JSON (`--file`); `$S` =
`.claude/skills/character-generation/scripts`.

Stop if the JSON's `class` is not `Magic-User`.

Everything a Magic-User needs at level 1 lives on three pages — read them
before recording anything:
`docs/Character/Classes/Magic-User/index.md` (class grants),
`docs/Character/Classes/Magic-User/Special Rules/Spells.md` (how the
spellbook works) and the career page in the JSON (`career_page`, its table
gives the daily spell slots).

## 1. Verify the class grant

- **No combat bonus.** Magic-Users get WS +0 / BS +0; `apply_class.py`
  leaves both at 0. Do not "fix" them.
- **Forced Chaos.** Alignment must be `Chaos`
  (docs/Character/Alignment.md); `apply_class.py` enforces it.
- **No armour, no shields.** If anything was bought, `finalize.py` prints a
  Magic-User armour-ban WARNING — resolve it (`buy.py remove` refunds), a
  Magic-User in armour cannot function.

## 2. Record the spellbook and starting spells

The spellbook holds **Read Magic plus three random first-level spells**
(class page → Starting Skills and Possessions; Spells.md → Beginning
Spells). Roll the three from the level-1 list (Read Magic is guaranteed and
never one of the three):

```
python3 - <<'PY'
import random, pathlib
d = pathlib.Path("docs/Character/Classes/Magic-User/Spells/Level 1")
pool = sorted(p.stem for p in d.glob("*.md") if p.stem != "Read Magic")
print("\n".join(random.sample(pool, 3)))
PY
```

The player also **defines the physical form of the spellbook** once, for
good (Spells.md: a bound book, clay tablets, an inscribed blade, tattoos on
the skin …). Interactive mode: offer a few forms fitting the concept and
let them pick; quick mode: choose one yourself. Record both:

```
python3 $S/annotate.py --file ... \
    --note "Spellbook (<form>): Read Magic, <spell 1>, <spell 2>, <spell 3>"
```

## 3. Record the daily spell slots

The career table's numbered columns are how many spells of each level may be
kept **memorized at once** — not what the spellbook holds. At level 1 every
Magic-User career shows **1st = 1**: one first-level spell prepared per day,
chosen from the spellbook after six hours' rest and one hour of study. Note
it so it survives to the sheet:

```
python3 $S/annotate.py --file ... --note "Prepared/day: 1 first-level spell (<career> table); pick from spellbook, casting expends it"
```

Casting has real teeth worth a one-line note if the player is new: no roll
to cast, but it takes the whole round and resolves at the *start of the next
round* — and any damage taken in between breaks concentration and wastes the
spell (Spells.md → Casting Spells).

## 4. Record the talents

From the class page: **Second Sight** and **Spellcasting Talent**; the
career grants **Read/Write Talent** at level 1 (career table, "New traits").
The current text also lists a bare **"6SS"** whose meaning it never spells
out — record it verbatim and flag it rather than inventing a mechanic:

```
python3 $S/annotate.py --file ... \
    --note "Talents: Second Sight, Spellcasting, Read/Write (career L1)" \
    --note "Class page also lists '6SS' — undefined shorthand in the current rulebook; confirm with the Referee"
```

## 5. Career specifics

Only two Magic-User careers are fully written; the rest are stubs. Read the
career page and act:

| Career | Setup |
| --- | --- |
| Wizard | Orthodox arcane scholar; no extra level-1 trait beyond Read/Write. Nothing to add. |
| Bright Wizard | Fire mage of the Imperial colleges. The signature **Lore of Aqshy Talent** arrives at level 2, not now — note it as a coming feature, don't grant it. |
| Witch / Grey Wizard / Light Wizard | Career pages are blank stubs (no Status, no traits). Record the concept as a note and tell the player the mechanics are pending rulebook content. |

**Status is blank on every Magic-User career page**, so `apply_class.py`
will have stopped for `--status`. Magic-Users are lettered and moneyed —
Silver is the sensible default, Gold for a well-placed college wizard; note
that you supplied it.

## 6. Re-render

```
python3 $S/finalize.py --file ...        # only if equipment changed
python3 $S/render_sheet.py --file ... --out characters/<slug>.md
```

Spells and talents render under the sheet's Notes section. When presenting
the character, say in one line what they can actually do turn one: one
prepared first-level spell (name it), a spellbook of four, no armour, and
fragile — the power is all in the spell.

# Rogue class kit

Finishes a Rogue after the core `character-generation` workflow. Works on the
same character JSON (`--file`); `$S` =
`.claude/skills/character-generation/scripts`.

Stop if the JSON's `class` is not `Rogue`.

Read `docs/Character/Classes/Rogue/index.md`, the career page (`career_page`)
and `docs/Adventuring/Skills.md` (what each skill does and how it is rolled).

## 1. What the Rogue class gives

From the class page: Wounds `1d6 (min 4)`, Stamina `1d3`, its own saving-throw
row (all set by `apply_class.py`), **WS +0 / BS +0**, no class Skill-Point
column, and an empty starting-possessions list. The class has no forced
alignment and its flavour is the adaptable outsider — "a Rogue may have been
anything before taking to the road." Everything with real mechanics is carried
by the *career*.

**Only the Thief career is fully written.** It is the game's skill specialist
(the "criminal"): see §2. The other five — Charlatan, Grave Robber, Outlaw,
Smuggler, Wrecker — are still blank stubs: see §3.

## 2. Thief — the skill specialist

The Thief page (now filled) sets Status **Brass**, a nine-skill list, the
**Sneak Attack** trait, and **2 Skill Points** at level 1 (`career_trait` and
`career_skills` are already in the JSON; the trait prints on the sheet). The
class table has no Skill-Point column, so `apply_class.py` leaves
`skill_points` at 0 — the 2 points come from the career and you assign them
here.

**Base proficiency.** Every character already has a 1-in-6 chance in each of
the Thief's skills (Architecture, Bushcraft, Climb, Doors, Languages, Search,
Sleight of Hand, Stealth, Tinkering). Skill Points raise them: one point =
+1-in-6 to one skill.

**Spend the 2 points.** Two ways, agent's judgement:

- **Given a brief** — if the request named a skill set or described the Thief
  ("a cat burglar", "a con artist", "a tomb robber"), assign the 2 points to
  fit it. A burglar leans Stealth/Climb/Doors; a pickpocket Sleight of
  Hand/Stealth; a tomb robber Search/Tinkering/Architecture.
- **No brief** — roll a weighted spread, the street-skills weighted high and
  the scholarly ones ("lore of magic" types) low, per the player's steer:

```
python3 - <<'PY'
import random
weights = {"Stealth":5, "Sleight of Hand":4, "Search":4, "Climb":3,
           "Doors":3, "Tinkering":3, "Bushcraft":2, "Architecture":1,
           "Languages":1}
POINTS, CAP = 2, 2          # 2 points at L1; both may pile on one skill (-> 3-in-6)
adv, skills, w = {}, list(weights), list(weights.values())
while sum(adv.values()) < POINTS:
    s = random.choices(skills, weights=w)[0]
    if adv.get(s, 0) < CAP:
        adv[s] = adv.get(s, 0) + 1
for s in sorted(adv):
    print(f"{s}={1 + adv[s]}")     # final N-in-6 = base 1 + advances
PY
```

Record the improved skills at their final rating (the untouched ones stay at
the universal 1-in-6, no need to list them):

```
python3 $S/annotate.py --file ... --skill "Stealth=2" --skill "Sleight of Hand=2"
python3 $S/annotate.py --file ... --note "Base 1-in-6 in all other Thief skills; +2 Skill Points per level after 1st"
```

**Sneak Attack.** The trait already prints, but note its trigger so it is not
forgotten in play: an attack on an unaware or Surprised target (classically
straight out of Stealth) rolls the weapon's damage dice twice and adds both.
The Thief also ignores the Climb-while-encumbered restriction — worth a note if
they are carrying much:

```
python3 $S/annotate.py --file ... --note "Sneak Attack: double damage dice vs unaware/Surprised targets. May Climb while encumbered."
```

## 3. The stub careers (Charlatan, Grave Robber, Outlaw, Smuggler, Wrecker)

These pages are blank — no Status, no skills, no trait. Don't invent crunch:

- `apply_class.py` will have stopped for `--status`; a Rogue has "no official
  social standing," so **Brass** is the honest default (Silver for one with
  some means). Note that you supplied it.
- Record the concept as a note and say the career's skills/trait are pending
  rulebook content — or, if the player wants it to play like the finished
  criminal, run it on the Thief chassis from §2 and say so.

```
python3 $S/annotate.py --file ... \
    --note "Status set to <Brass|Silver> by judgement — this Rogue career defines none yet" \
    --note "<Career>: <one-line concept>; career skills/trait are pending rulebook content"
```

## 4. Mind the WS/BS 0 weapon constraint

With WS +0 and BS +0 a Rogue reliably uses only **Basic** weapons. Trained
melee weapons need WS +1, most ranged need BS +1, and firearms always need
BS +1 — none of which a level-1 Rogue has. If `finalize.py` warned about an
unusable weapon, swap it for a Basic one (dagger, club, sling/thrown as the
tables allow); a Thief's edge is the Sneak Attack, not the weapon list.

## 5. Re-render

```
python3 $S/finalize.py --file ...        # only if equipment changed
python3 $S/render_sheet.py --file ... --out characters/<slug>.md
```

Skills, Sneak Attack and any notes render on the sheet. Present the Thief in a
line: a Brass-status specialist who lives by nine skills and a doubling Sneak
Attack from the shadows — light on armour and weapons, heavy on guile.
